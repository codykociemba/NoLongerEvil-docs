## What is Self-Hosting?

Self-hosting means running the entire NoLongerEvil stack on your own infrastructure rather than using our managed service. Your thermostat connects to **your server**, giving you complete control over data and functionality.

<Warning>
  **Advanced users only!** Self-hosting requires:
  - Server administration experience
  - Comfort with command line/terminal
  - Understanding of networking, Docker, and databases
  - Time for setup and ongoing maintenance

  If this sounds overwhelming, we recommend the [hosted service](/hosted/overview).
</Warning>

## Architecture Overview

When self-hosting, you'll run these components:

<Steps>
  <Step title="API Server (Port 443)">
    Node.js HTTPS server that intercepts thermostat communication

    **Responsibilities**:
    - Handle device connections and authentication
    - Proxy weather requests
    - Manage device state synchronization
    - Generate entry keys

    **Requirements**:
    - Public IP or DDNS
    - SSL/TLS certificate
    - Port 443 accessible
  </Step>

  <Step title="Control API (Port 8081)">
    HTTP API for sending commands and querying device status

    **Responsibilities**:
    - Accept control commands (temperature, mode)
    - Provide device status
    - List connected devices

    **Requirements**:
    - Port 8081 accessible (can be localhost-only)
  </Step>

  <Step title="Database (Convex)">
    Backend database for persistent storage

    **Stores**:
    - Device state and configuration
    - User accounts (if using frontend)
    - Entry keys
    - Weather cache

    **Options**:
    - Convex Cloud (easiest)
    - Self-hosted Convex (advanced)
  </Step>

  <Step title="Frontend (Optional)">
    Next.js web application for device management

    **Features**:
    - User authentication (Clerk)
    - Device dashboard
    - Control interface

    **Note**: You can use the hosted frontend at nolongerevil.com pointed at your self-hosted API
  </Step>
</Steps>

## What You'll Need

### Technical Requirements

<Checklist>
  - [ ] Server or VPS running Linux (Ubuntu 22.04+ recommended)
  - [ ] Docker and Docker Compose installed
  - [ ] Domain name (optional but recommended)
  - [ ] SSL certificate (Let's Encrypt or self-signed)
  - [ ] Port 443 accessible from internet
  - [ ] Port 8081 accessible (can be localhost or internal network)
</Checklist>

### Technical Skills

<Checklist>
  - [ ] Command line / terminal navigation
  - [ ] Basic networking (DNS, ports, firewalls)
  - [ ] Docker containers and docker-compose
  - [ ] Environment variables and configuration files
  - [ ] SSL/TLS certificate management
  - [ ] (Optional) Next.js and Node.js for customization
</Checklist>

### Estimated Time

- **Initial Setup**: 2-4 hours
- **Troubleshooting**: 1-2 hours (varies)
- **Ongoing Maintenance**: ~1 hour/month

## Hybrid Approach (Recommended for Beginners)

You don't have to self-host everything:

<Accordion title="Option 1: Self-Hosted API + Hosted Frontend">
  **What you run**: API server only

  **What NoLongerEvil provides**: Web dashboard

  **How it works**:
  - Your thermostat connects to your API server
  - You configure the hosted frontend to point to your API URL
  - You control your data while using our interface

  **Pros**:
  - Easiest self-hosted option
  - Data ownership
  - No frontend maintenance

  **Cons**:
  - Depend on our frontend availability
  - Must expose API publicly
</Accordion>

<Accordion title="Option 2: Full Self-Hosted">
  **What you run**: API server + frontend + database

  **What NoLongerEvil provides**: Nothing (fully independent)

  **Pros**:
  - Complete independence
  - Full customization
  - No external dependencies

  **Cons**:
  - Most complex setup
  - Frontend maintenance required
  - Authentication setup needed
</Accordion>

## Challenges to Expect

<Warning>
  Be prepared to troubleshoot:

  - Docker networking and container issues
  - SSL certificate setup and renewal
  - Port forwarding and firewall configuration
  - DNS and domain management
  - Database connection problems
  - Environment variable configuration errors
